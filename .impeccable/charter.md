# Impeccable Charter

## Purpose

This workspace is initialized for careful, auditable implementation. The goal is to make each change easy to understand, verify, and continue.

## Working Principles

- Start from the current repo state and preserve user changes.
- Prefer the existing project patterns once a codebase exists.
- Keep changes scoped to the requested outcome.
- Record meaningful assumptions and decisions in `.impeccable/tasks.md`.
- Verify with the narrowest useful check first, then broaden when risk warrants it.
- Call out any skipped verification clearly in the handoff.

## Definition of Done

A change is done when the requested behavior is implemented, relevant files are updated, verification has run or been explicitly deferred, and the handoff explains the result plainly.
